### 1. Core Goal & Philosophy
Build an emergent, self-learning trading agent on Solana that:
- Discovers and analyzes high-performing wallets (their patterns, risk management, timing, "why" they win).
- Evolves from passive copy-trading → active understanding → autonomous low-risk, high-ROI strategy generation.
- Emphasizes **ethical, non-adversarial execution**: no frontrunning (no MEV bundles, Jito, ultra-low-latency private RPCs), no sniping (no sub-second race for new launches).
- Leverages the agent's own speed limitations (~seconds polling/inference on standard/public RPCs) as a feature: learn to be selectively patient, filtering out manipulated early chaos and entering only on persisting edges.

### 2. Key Principles & Constraints
- **Learning from full cycle**: Track wins + losses, metrics (hit rate, ROI, drawdown, streaks, "when not to trade").
- **Market realism**: Crypto (esp. Solana memecoins) has fleeting micro-edges; focus on cluster behaviors, timing, regime detection over naive next-trade prediction.
- **Efficiency**: Token-sparse LLM calls (summarize trades/clusters, trigger on key events), external lightweight metrics, eventual distillation to smaller/faster models for execution.
- **No hardcoded timing rules** — especially for Pump.fun: agent learns when to enter (or skip) based on historical analogs + explicit latency awareness in prompts/embeddings.

### 3. Data Representation & Processing
- **Hybrid inputs**:
  - Vectorizable: Trade size, PnL, leverage, time deltas, token pair, cluster ID, volume streaks, holder distribution, curve progress %, dev activity metrics.
  - Embeddings/text: Market sentiment (lightweight X/social pulls if feasible), inferred rationale, token metadata (meme strength), on-chain "story" summaries.
- **Clusters** for pattern mining: e.g., momentum runners, swing holds, risk-managed entries, Pump.fun-specific (organic post-1min momentum, avoid dev-dump signatures, slow-build graduates).

### 4. Data Splits & Feedback Loops
- Historical pre-training (past trades for clusters/embeddings).
- Simulation/sandbox (safe strategy testing, mistake learning without real funds).
- Live incremental updates (feed only critical events: high-impact PnL, drawdowns, rugs → refine weights, embeddings, policies).
- Strong anti-overfitting: regime detection, forgetting mechanisms, penalize regime-mismatched trades heavily.

### 5. Multi-Agent Architecture (with Dedicated Pump.fun Handling)
- **Multiple specialized agents** explore different clusters/strategies.
- **Supervisory agent** aggregates outputs, allocates capital, enforces portfolio risk (drawdown halts, correlation awareness, position sizing).
- **Dedicated PumpLaunchAgent** (or small sub-cluster): 
  - Subscribes to new Pump.fun pairs via standard websocket/RPC (public endpoints — no ultra-fast feeds).
  - Staged evaluation on detected launches (thousands daily, but focus on high-signal subset, e.g., from copied wallets or cluster matches):
    - Stage 1 (lightweight/external): Immediate filters (dev sells? extreme concentration? bot-heavy early buys? → skip).
    - Stage 2 (sparse LLM trigger on promising signals): Prompt includes vectors/embeddings (age, curve %, volume/holders streaks, metadata), latency awareness ("execution ~5–15s; only enter if edge likely persists post-delay"), historical analogs from winners/graduates.
    - Agents specialize within: e.g., one for "post-shakeout momentum", one for "community-driven slow build".
  - Learns from outcomes: e.g., "Entered at 2min on volume confirmation → +ROI if no rug" vs. "Early entry without div → rekt → heavy penalty".
  - Capital ring-fenced (e.g., max 5–10% portfolio to Pump launches), stricter halts due to high variance.
- Emergent behavior: Faster pattern discovery across agents; supervisor merges cross-knowledge.

### 6. Pump.fun Specifics (Current 2026 Meta Context)
- **Regime**: ~20k–30k+ daily launches; graduation rate (to Raydium/PumpSwap at ~$69k–$100k MC / ~80 SOL in curve) recently spiked >1% (highest since summer 2025, e.g., 269 grads in a day from high-volume periods), but historically <1% long-term; most fail/rug fast.
- **Non-snipe niche**: Ultra-early (0–10s) dominated by bots/snipers; agent targets mid-stage confirmation (30s–10+ min): sustained volume without dumps, organic holders, no red flags — where edges can persist longer than agent's latency.
- **Learned behavior**: Skip 90%+ launches; enter selectively on high-conviction patterns (e.g., post-rug filter, momentum streaks) if projected ROI justifies risk given latency.

### 7. Execution & Infra Guardrails
- Poll standard RPCs (Helius/QuickNode public) + websockets for tx/launch detection.
- No priority fee spiking, bundles, or MEV tools — keeps it clean/ethical.
- Entry decisions: Supervisory approval based on agent conviction + portfolio state.
- Post-entry: Adaptive exits (learned trailing/stops), tiny initial sizes scaling with conviction.
